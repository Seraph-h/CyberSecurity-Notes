## 一、内网渗透简介

### 1.1 什么是内网渗透

内网渗透（Intranet Penetration）是指在获取目标外网入口（如Web漏洞、钓鱼邮件）后，进入目标内部网络，进行进一步的权限提升、横向移动、数据窃取等操作。

```
外网入口（GetShell） -> 内网信息收集 -> 权限提升 -> 横向移动 -> 域控 -> 数据导出
```

### 1.2 为什么面试要问内网渗透

| 考察点 | 面试官想知道什么 |
|--------|----------------|
| 实战经验 | 你是否做过完整的内网渗透项目 |
| 工具使用 | Cobalt Strike、Metasploit、BloodHound 等 |
| 域渗透 | 是否理解 Active Directory、Kerberos |
| 防御思路 | 你知道怎么防，才能知道怎么攻 |

### 1.3 内网渗透 vs 外网渗透

| 对比项 | 外网渗透 | 内网渗透 |
|--------|----------|----------|
| 目标 | 公网暴露的服务 | 内部网络主机 |
| 网络环境 | 可直接访问 | 需要跳板/隧道 |
| 工具 | Nmap、SQLMap、BurpSuite | Cobalt Strike、Metasploit、BloodHound |
| 权限 | 普通用户/服务权限 | 域用户/本地管理员/域管理员 |
| 关注点 | 漏洞利用 | 凭证窃取、横向移动 |

### 4.6 横向移动 - Pass-the-Hash

**获取Hash（Mimikatz）**:
```cmd
privilege::debug
sekurlsa::logonpasswords
```

**使用Hash登录**:
```bash
# CrackMapExec
crackmapexec smb 192.168.1.0/24 -u administrator -H NTLM_HASH

# Impacket-psexec
impacket-psexec -hashes LMHASH:NTHASH administrator@192.168.1.10

# Impacket-wmiexec
impacket-wmiexec -hashes LMHASH:NTHASH administrator@192.168.1.10
```

*说明：*Pass-the-Hash 需要 Mimikatz 提取 NTLM Hash，然后用 CrackMapExec 或 Impacket 传递。当前环境 Mimikatz 未安装，且 Windows 7 TLS 限制无法直接下载。实际渗透中，先用 Mimikatz `sekurlsa::logonpasswords` 提取 Hash，再用 `crackmapexec smb IP -u user -H Hash` 或 `impacket-psexec -hashes Hash user@IP` 横向移动。

## 五、Cobalt Strike简介

### 5.1 什么是Cobalt Strike

Cobalt Strike是红队（Red Team）常用的商业渗透测试工具，基于Metasploit，但功能更强大，支持团队协作、隐蔽通信、持久化等。

### 5.2 核心功能

| 功能 | 说明 |
|------|------|
| Beacon | 后门程序，支持HTTP/HTTPS/DNS/SMB通信 |
| Team Server | 团队协作服务器 |
| Aggressor Script | 自动化脚本 |
| Malleable C2 | 自定义通信特征，绕过流量检测 |
| 横向移动 | psexec、wmi、winrm等 |
| 权限提升 | UAC绕过、Token窃取 |

### 5.3 登录

```
1. 启动 Team Server：./teamserver <IP> <密码>
2. 启动客户端：./cobaltstrike.sh（或 java -jar cobaltstrike-client.jar）
3. 登录界面填写：
   - Host: Team Server IP
   - Port: 50050
   - User: 任意用户名
   - Password: Team Server 密码
4. 登录成功后界面：
   - 左侧：Targets（目标列表）
   - 中间：Cobalt Strike 图形化操作区
   - 右侧：Beacon 会话列表
   - 底部：命令控制台
```

## 六、内网渗透对比总结

### 6.1 面试口述框架

> "内网渗透流程: 首先通过外网漏洞获取入口（GetShell），然后进行主机信息收集（系统、网络、进程、用户），接着进行权限提升（Windows用Windows-Exploit-Suggester/PowerUp，Linux用linux-exploit-suggester/SUID），之后进行横向移动（Pass-the-Hash、WMI、SMB、SSH隧道），最终目标是域控（Domain Controller）。常用工具: Cobalt Strike、Metasploit、Mimikatz、BloodHound、CrackMapExec。"

### 6.2 核心工具速查表

| 工具 | 用途 | 场景 |
|------|------|------|
| Cobalt Strike | 红队框架 | 团队协作、隐蔽通信 |
| Metasploit | 渗透测试框架 | 漏洞利用、后渗透 |
| Mimikatz | 凭据提取 | 获取明文密码/Hash/Ticket |
| BloodHound | 域关系分析 | 可视化域内权限关系 |
| CrackMapExec | 批量横向移动 | SMB/WMI/WinRM批量执行 |
| PowerView | 域信息收集 | PowerShell域内信息收集 |
| PowerUp | Windows提权 | 检查提权漏洞 |
| Windows-Exploit-Suggester | Windows漏洞分析 | 根据补丁找提权漏洞 |
| linux-exploit-suggester | Linux漏洞分析 | 根据内核找提权漏洞 |
| Impacket | 协议工具集 | psexec、smbexec、wmiexec |

### 6.3 核心参数速记口诀

```
外网打点到内网，信息收集是首要
Windows用systeminfo，Linux用uname -a
提权先看补丁号，Exploit-Suggester跑一跑
横向移动靠Hash，Mimikatz提取不能少
域控最终是目标，BloodHound画关系图
Cobalt Strike是神器，红队渗透离不了
```

## 八、面试高频题

### Q1: 内网渗透的流程是什么?

> "外网入口 -> 主机信息收集 -> 权限提升 -> 横向移动 -> 域控。信息收集包括系统信息、网络配置、用户权限、进程服务; 提权用系统漏洞或服务配置错误; 横向移动用Pass-the-Hash、WMI、SMB等; 最终目标是获取域管理员权限。"

### Q2: 什么是Pass-the-Hash?

> "Pass-the-Hash（哈希传递）是指使用NTLM Hash直接进行身份认证，无需明文密码。Windows网络认证使用Challenge-Response机制，攻击者获取Hash后可以直接用于认证。工具: Mimikatz提取Hash，CrackMapExec或Impacket进行传递。"

### Q3: Windows和Linux提权各有哪些方法?

> "Windows: 系统漏洞提权（Windows-Exploit-Suggester）、服务配置错误（PowerUp）、计划任务提权、内核漏洞。Linux: SUID提权（find / -perm -4000）、Sudo配置错误（sudo -l）、内核漏洞（linux-exploit-suggester）、计划任务提权。"

### Q4: Cobalt Strike和Metasploit有什么区别?

> "Metasploit是开源渗透测试框架，功能全面但隐蔽性一般。Cobalt Strike是商业红队工具，基于Metasploit但增加了团队协作、隐蔽通信（Malleable C2）、持久化、报告生成等功能，更适合高级持续威胁（APT）模拟。"

### Q5: 什么是BloodHound?

> "BloodHound是域环境分析工具，通过收集Active Directory数据，可视化展示域内用户、组、计算机的权限关系，帮助攻击者找到最短路径获取域管理员权限（如ACL滥用、Kerberoasting路径）。"

### Q6: 横向移动有哪些方法?

> "常用方法: Pass-the-Hash（使用NTLM Hash认证）、Pass-the-Ticket（使用Kerberos Ticket）、WMI执行（wmic/impacket-wmiexec）、SMB执行（PsExec/impacket-smbexec）、远程桌面（RDP）、SSH隧道。"
