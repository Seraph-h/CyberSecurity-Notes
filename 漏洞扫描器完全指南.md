## 一、漏洞扫描器简介

### 1.1 什么是漏洞扫描器

**漏洞扫描器**是自动化检测目标系统安全漏洞的工具，通过：

- 发送特定 Payload 测试已知漏洞
- 比对服务版本与漏洞数据库
- 检查配置缺陷和弱口令
- 生成详细的漏洞报告

### 1.2 常见漏洞扫描器

| 扫描器 | 特点 | 适用场景 |
|--------|------|---------|
| **Nessus** | 商业工具，漏洞库最全，报告专业 | 企业渗透测试、合规检查 |
| **OpenVAS** | 开源免费，Nessus 前身 | 个人学习、小型项目 |
| **AWVS** | Web 漏洞扫描专业 | Web 应用安全测试 |
| **Xray** | 国产，被动扫描，无爬虫 | 渗透测试辅助 |
| **Goby** | 国产，资产测绘 + 漏洞扫描 | 红队快速打点 |

### 1.3 为什么面试要问漏洞扫描器

| 考察点 | 面试官想知道什么 |
|--------|---------------|
| 工具使用 | 你是否会用专业工具提高效率 |
| 结果分析 | 你能从扫描结果中筛选有效漏洞 |
| 误报处理 | 你知道扫描器有假阳性 |
| 合规意识 | 你了解企业安全合规要求 |

### 1.4 漏洞扫描器 vs 手动渗透

| 对比项 | 漏洞扫描器 | 手动渗透 |
|--------|-----------|---------|
| 速度 | 快，自动化 | 慢，逐条测试 |
| 覆盖面 | 广，基于漏洞库 | 深，针对特定目标 |
| 误报率 | 高 | 低 |
| 逻辑漏洞 | 无法发现 | 可以发现 |
| 0day漏洞 | 无法发现 | 可能发现 |
| 面试要求 | 必须会（效率） | 必须会（深度） |
| 实际工作 | 快速发现已知漏洞 | 深入挖掘复杂漏洞 |

**面试一句话**："扫描器快速发现已知漏洞，手动渗透深入挖掘逻辑漏洞和0day，两者结合才是完整的渗透测试流程。"

## 二、OpenVAS 安装与使用

### 2.1 OpenVAS 简介

OpenVAS（Open Vulnerability Assessment System）是 Nessus 的开源分支，完全免费。

### 2.2 安装方式

#### Kali Linux 自带

```bash
# 更新系统
sudo apt-get update

# 安装 OpenVAS
sudo apt-get install openvas

# 初始化配置
sudo gvm-setup

# 启动服务
sudo gvm-start

# 查看管理员密码
sudo gvm-check-setup
```

#### Docker 安装（推荐）

```bash
# 拉取镜像
docker pull mikesplain/openvas

# 运行容器
docker run -d -p 443:443 --name openvas mikesplain/openvas

# 访问 https://localhost
# 默认账号：admin / 密码：admin
```

## 三、Nessus 使用

![78465364874](漏洞扫描器完全指南.assets/1784653648741.png)

### 3.1 Nessus 简介

Nessus 是全球最流行的商业漏洞扫描器，漏洞库最全，报告最专业。

### 3.2 安装

```bash
# 下载安装包
wget https://www.tenable.com/downloads/api/v1/public/pages/nessus/downloads/xxxx

# 安装
sudo dpkg -i Nessus-xxxx.deb

# 启动服务
sudo systemctl start nessusd

# 访问 https://localhost:8834
# 激活码需官网申请（免费版有16个IP限制）
```

### 3.3 创建扫描

1. 点击 New Scan
2. 选择扫描模板：
   - **Basic Network Scan**：基础网络扫描
   - **Advanced Scan**：高级扫描
   - **Web Application Tests**：Web应用扫描
   - **Malware Scan**：恶意软件扫描


3. 配置扫描参数：
   - Name：任务名称
   - Targets：目标IP/域名
   - Schedule：定时扫描（可选）


4. 启动扫描

### 3.4 查看结果

**导出报告**：

1. 点击 Export
2. 选择格式：
   - PDF（给客户）
   - HTML（给技术）
   - CSV（给数据分析）
   - Nessus XML（导入其他工具）

## 四、扫描结果分析

### 4.1 如何筛选有效漏洞

| 筛选条件 | 说明 |
|---------|------|
| **CVSS 评分 >= 7.0** | 高危及以上 |
| **有公开 EXP** | 可验证、可利用 |
| **可远程利用** | 无需本地权限 |
| **业务相关** | 影响核心业务系统 |

### 4.2 常见误报处理

| 误报类型 | 原因 | 验证方法 |
|---------|------|---------|
| **版本误报** | 扫描器基于版本匹配，实际已打补丁 | 手动验证 |
| **配置误报** | 某些配置被误判为漏洞 | 查看详细说明 |
| **网络误报** | 中间设备干扰 | 直接访问目标验证 |

## 五、Xray 被动扫描（国产推荐）

### 5.1 Xray 简介

Xray 是长亭科技开发的安全评估工具，特点是**被动扫描**，不主动爬虫，通过代理方式检测漏洞。

### 5.2 安装

```bash
# 下载
wget https://github.com/chaitin/xray/releases/download/xxx/xray_linux_amd64.zip

# 解压
unzip xray_linux_amd64.zip

# 生成证书
./xray genca

# 启动代理扫描
./xray webscan --listen 127.0.0.1:7777 --html-output result.html
```

### 5.3 使用

1. 浏览器设置代理：127.0.0.1:7777
2. 导入 Xray 证书
3. 正常浏览目标网站
4. Xray 自动检测漏洞

## 六、漏洞扫描器对比总结

### 6.1 面试口述框架

**Q：OpenVAS 和 Nessus 怎么选？**

> "OpenVAS 开源免费，适合个人学习和小型项目。Nessus 商业工具，漏洞库最全，报告最专业，适合企业渗透测试和合规检查。实际工作中，预算充足选 Nessus，预算有限选 OpenVAS，Web 渗透配合 Xray 被动扫描。"

**Q：扫描结果有很多漏洞，怎么确定优先级？**

> "三步筛选：1. 按 CVSS 评分排序，优先 Critical 和 High；2. 筛选有公开 EXP 的漏洞，可验证可利用；3. 结合业务重要性，核心业务系统优先。最后手动验证排除误报。"

**Q：漏洞扫描有什么局限性？**

> "五个局限：1. 误报率高，需要人工验证；2. 无法发现逻辑漏洞（如越权、0元购买）；3. 无法发现0day漏洞（没有插件）；4. 扫描可能触发防护设备（WAF/IDS）；5. 大型网络扫描耗时长。扫描器是辅助工具，不能替代人工渗透。"

### 6.2 核心参数速记口诀

## 八、面试高频题

**Q1：OpenVAS 和 Nessus 有什么区别？**

> "OpenVAS 是 Nessus 的开源分支，完全免费，适合个人学习和小型项目。Nessus 是商业工具，漏洞库更全面，报告更专业，适合企业渗透测试和合规检查。两者核心功能相似，都是基于插件的漏洞扫描。"

**Q2：漏洞扫描器的原理是什么？**

> "漏洞扫描器通过三种方式检测漏洞：1. 版本匹配，比对服务版本与已知漏洞数据库；2. 漏洞验证，发送特定 Payload 测试漏洞是否存在；3. 配置审计，检查系统配置是否符合安全基线。扫描结果可能有误报，需要人工验证。"

**Q3：扫描结果有很多漏洞，怎么确定优先级？**

> "按 CVSS 评分排序，优先处理 Critical 和 High 级别。然后筛选有公开 EXP 的漏洞（可验证可利用），再筛选可远程利用的（影响范围大）。最后结合业务重要性，核心业务系统的漏洞优先修复。"

**Q4：漏洞扫描有什么局限性？**

> "1. 误报率高，需要人工验证；2. 无法发现逻辑漏洞（如越权、0元购买）；3. 无法发现0day漏洞（没有插件）；4. 扫描可能触发防护设备（WAF/IDS）；5. 扫描大型网络耗时长。"

**Q5：Xray 的被动扫描和 Nessus 的主动扫描有什么区别？**

> "主动扫描（Nessus）是扫描器主动发送请求探测目标，覆盖面广但可能触发防护。被动扫描（Xray）是通过代理方式，只检测用户浏览过程中触发的请求，不主动爬虫，隐蔽性高但覆盖面依赖用户操作。两者结合使用效果最好。"

**Q6：扫描时怎么避免触发 WAF/IDS？**

> "1. 降低扫描速度（Nessus 的 Scan Low Bandwidth）；2. 分时段扫描（业务低峰期）；3. 使用白名单 IP（提前通知安全团队）；4. 配合 Nmap 先探测，再针对性扫描；5. 使用 Xray 被动扫描替代主动扫描。"
